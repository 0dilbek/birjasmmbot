# BirjaSMM — Telegram-бот для биржи SMM-услуг

[O'zbekcha versiyasi (README_UZ.md)](README_UZ.md)

Telegram-бот для соединения **заказчиков** (клиентов) и **исполнителей** в сфере SMM, видеопродакшна и блогерства. Поддерживает два языка интерфейса: **русский** и **узбекский**.

## Конфигурация и запуск

### 1. Скопировать и заполнить `.env`

```bash
cp .env.example .env
```

```env
BOT_TOKEN=ваш_токен_от_@BotFather
DATABASE_URL=postgres://user:password@localhost:5432/birjasmm?sslmode=disable
ADMIN_IDS=123456789,987654321   # Telegram ID администраторов через запятую
FREE_RESPONSES=5                # Бесплатных откликов для исполнителя
```

### 2. Создать базу данных

```bash
createdb birjasmm
```

Миграции применяются **автоматически** при старте бота (все `.sql` файлы из папки `migrations/` по порядку).

### 3. Запустить бота

```bash
go run ./cmd/bot/
# или
go build -o birjasmm ./cmd/bot/ && ./birjasmm
# или через Makefile
make run
```

## Роли пользователей

| Роль | Может |
|------|-------|
| **Заказчик** (`client`) | Создавать задачи, выбирать исполнителя, завершать задачу, оставлять отзыв, верифицироваться |
| **Исполнитель** (`executor`) | Откликаться на задачи, покупать подписку, проходить верификацию, просматривать активные заказы |
| **Администратор** | Управлять пользователями, задачами, верификациями, статистикой, подтверждать платежи |

> Администраторы определяются через `ADMIN_IDS` в `.env`. Их интерфейс всегда на **русском языке**.

## Сценарии

### 1. Новый пользователь

**Команда:** `/start`

1. Бот проверяет, есть ли пользователь в БД.
2. Если нет — показывает **выбор языка**: 🇷🇺 Русский / 🇺🇿 O'zbek.
3. Пользователь нажимает кнопку → язык сохраняется в FSM (до регистрации) и затем в БД.
4. Показывается **выбор роли**: Заказчик / Исполнитель.

```
/start
  └── Новый пользователь?
        └── Выбор языка (RU / UZ)
              └── Выбор роли (Заказчик / Исполнитель)
                    └── Регистрация...
```

---

### 2. Регистрация заказчика

**Триггер:** выбор роли "Заказчик"

Шаги FSM:

| Шаг | Состояние | Что вводит пользователь |
|-----|-----------|------------------------|
| 1 | `StateRegName` | Имя (текст) |
| 2 | `StateRegPhone` | Телефон (кнопка "Поделиться контактом") |
| 3 | `StateRegCity` | Город (текст) |
| 4 | `StateRegBusinessName` | Название бизнеса / ИП (текст) |

После завершения:
- Создаётся запись в `users` и `client_profiles`.
- Открывается **главное меню заказчика**:
  - ➕ Создать задачу
  - 📋 Мои задачи
  - 👤 Профиль
  - 🔍 Исполнители
  - ✅ Верификация
  - 🌐 Сменить язык

---

### 3. Регистрация исполнителя

**Триггер:** выбор роли "Исполнитель"

Шаги FSM:

| Шаг | Состояние | Что вводит пользователь |
|-----|-----------|------------------------|
| 1 | `StateRegName` | Имя |
| 2 | `StateRegPhone` | Телефон |
| 3 | `StateRegCity` | Город |
| 4 | `StateRegCategory` | Категория (кнопки: SMM / Видео / Блогер) |
| 5 | `StateRegExp` | Опыт в годах (число) |
| 6 | `StateRegDesc` | Описание услуг (текст) |
| 7 | `StateRegPortfolio` | Ссылки на портфолио (текст или "Пропустить") |

После завершения:
- Создаётся `users` + `executor_profiles`.
- Открывается **главное меню исполнителя**:
  - 🔍 Найти заказы
  - 📂 Мои отклики
  - ⚙️ В работе
  - 💳 Подписка
  - 👤 Профиль
  - 🌐 Сменить язык

---

### 4. Создание задачи

**Доступно:** только заказчикам  
**Кнопка:** ➕ Создать задачу

Шаги FSM:

| Шаг | Состояние | Ввод | Ограничение |
|-----|-----------|------|-------------|
| 1 | `StateTaskTitle` | Название задачи | Макс. 50 символов |
| 2 | `StateTaskDescription` | Описание | Макс. 300 символов |
| 3 | `StateTaskCategory` | Категория (кнопки) | SMM / Видео / Блогер |
| 4 | `StateTaskBudgetType` | Тип бюджета | Фиксированный / Диапазон / Договорной |
| 5 | `StateTaskBudgetFrom` | Сумма (или мин. сумма) | Только для fixed/range |
| 6 | `StateTaskBudgetTo` | Макс. сумма | Только для range |
| 7 | `StateTaskDeadline` | Дедлайн | Формат: `ДД.ММ.ГГГГ` |
| 8 | `StateTaskRefs` | Референсы / ссылки | Можно пропустить |
| 9 | `StateTaskUrgent` | Срочная задача? | Да / Нет |
| 10 | — | Подтверждение | Опубликовать / Редактировать / Отмена |

После публикации:
- Задача сохраняется со статусом `open`.
- **Уведомление** отправляется всем исполнителям подходящей категории.
- Если задача **срочная** (`is_urgent = true`) — уведомление получают только исполнители с подпиской **PRO**.

---

### 5. Просмотр задач и отклик

**Кнопка (исполнитель):** 🔍 Найти заказы

- Показывает открытые задачи с фильтром по категории исполнителя.
- Если нет подписки PRO — срочные задачи скрыты.
- Пагинация: кнопка "Следующая страница".
- Можно фильтровать по категории и минимальному бюджету.

**Отклик на задачу:**

| Шаг | Состояние | Ввод |
|-----|-----------|------|
| 1 | `StateRespondMsg` | Сообщение заказчику (текст) |
| 2 | `StateRespondPrice` | Предлагаемая цена (число или "Пропустить") |

Ограничения:
- Один исполнитель — один отклик на задачу.
- Макс. 15 откликов на задачу (`max_responses`).
- Без подписки — лимит бесплатных откликов (по умолчанию 5).
- После отклика заказчику приходит уведомление.

После отклика исполнитель может **отозвать** его пока отклик активен.

---

### 6. Выбор исполнителя

**Кнопка (заказчик):** 📋 Мои задачи → просмотр откликов

- Заказчик видит список откликов: имя, рейтинг, цена, сообщение.
- Нажимает "Выбрать" рядом с нужным исполнителем.
- Остальные отклики автоматически **отклоняются** (исполнители получают уведомление).
- Выбранный исполнитель получает уведомление с контактами заказчика.
- Задача переходит в статус `in_progress`.

---

### 7. Завершение задачи и отзыв

**Кнопка (заказчик):** 📋 Мои задачи → завершить

- Задача переходит в статус `completed`.
- Заказчику предлагается **оставить отзыв** об исполнителе.

Шаги отзыва:

| Шаг | Состояние | Ввод |
|-----|-----------|------|
| 1 | `StateReviewRating` | Оценка 1–5 (кнопки с эмодзи) |
| 2 | `StateReviewComment` | Текстовый комментарий (или "Пропустить") |

После отзыва:
- Рейтинг исполнителя пересчитывается в БД (`UpdateRating`).

---

### 8. Верификация исполнителя

**Кнопка (исполнитель):** через меню или при попытке отклика

**Цель:** получить галочку ✅ в профиле (доверие заказчиков).

Шаги:

| Шаг | Состояние | Ввод |
|-----|-----------|------|
| 1 | — | Нажать "Начать верификацию" |
| 2 | `StateVerifVideo` | Видео с произнесением кода (4 цифры от TelegramID) |

После отправки видео:
- Статус верификации → `pending`.
- Администраторам приходит уведомление.

Администратор нажимает **Одобрить** или **Отклонить**:
- **Одобрить** → `is_verified = true` в `executor_profiles`, пользователю уведомление.
- **Отклонить** → статус сбрасывается в `none`, пользователю уведомление.

---

### 9. Верификация заказчика

**Кнопка (заказчик):** ✅ Верификация

**Цель:** подтвердить личность / юридический статус.

Шаги:

| Шаг | Состояние | Ввод |
|-----|-----------|------|
| 1 | — | Нажать "Начать верификацию" |
| 2 | `StateClientVerif` | Фото/скан документа (фото или файл) |

После отправки:
- Статус → `pending`.
- Администраторам приходит фото документа + кнопки "Верифицировать / Отклонить".
- После одобрения: `client_profiles.is_verified = true`.

---

### 10. Подписка и оплата

**Кнопка (исполнитель):** 💳 Подписка

Тарифы:

| Тариф | Цена | Возможности |
|-------|------|-------------|
| **BASIC** | 50 000 сум/мес | Безлимитные отклики |
| **PRO** | 100 000 сум/мес | Безлимитные отклики + срочные задачи + приоритет |

**Шаг оплаты:**

1. Бот показывает номер карты и сумму.
2. Исполнитель переводит деньги и отправляет **чек** (фото, файл или текст).
3. Чек и уведомление отправляются администраторам.

Администратор нажимает **Подтвердить** или **Отклонить**:
- **Подтвердить** → активируется подписка на 30 дней, флаг `is_pro = true` для PRO.
- **Отклонить** → исполнителю уведомление об отклонении.
- Сообщение у администраторов **обновляется** (редактируется) со статусом платежа.

---

### 11. Смена языка

**Кнопка:** 🌐 Сменить язык (в меню обеих ролей)

- Показывается выбор: 🇷🇺 Русский / 🇺🇿 O'zbek.
- Для зарегистрированных пользователей язык обновляется в БД.
- Для незарегистрированных — хранится в FSM до создания аккаунта.
- Все последующие сообщения бота приходят на выбранном языке.
- **Уведомления** (новые задачи, выбор исполнителя) отправляются каждому пользователю **на его языке**.

## FSM

Бот использует **Finite State Machine** (конечный автомат) для ведения многошаговых диалогов.

**Расположение:** `internal/bot/fsm/fsm.go`  
**Хранилище:** `sync.Map` (in-memory, сбрасывается при перезапуске)

### Состояния

| Константа | Описание |
|-----------|---------|
| `StateRegName` | Ожидание имени при регистрации |
| `StateRegPhone` | Ожидание номера телефона |
| `StateRegCity` | Ожидание города |
| `StateRegCategory` | Выбор категории (для исполнителя) |
| `StateRegExp` | Ввод опыта |
| `StateRegDesc` | Описание услуг |
| `StateRegPortfolio` | Ссылки на портфолио |
| `StateRegBusinessName` | Название бизнеса (для заказчика) |
| `StateTaskTitle` | Название задачи |
| `StateTaskDescription` | Описание задачи |
| `StateTaskCategory` | Категория задачи |
| `StateTaskBudgetType` | Тип бюджета |
| `StateTaskBudgetFrom` | Сумма / минимальная сумма |
| `StateTaskBudgetTo` | Максимальная сумма |
| `StateTaskDeadline` | Дедлайн |
| `StateTaskRefs` | Референсы |
| `StateTaskUrgent` | Срочность |
| `StateRespondMsg` | Сообщение в отклике |
| `StateRespondPrice` | Цена в отклике |
| `StateReviewRating` | Оценка отзыва |
| `StateReviewComment` | Текст отзыва |
| `StateVerifVideo` | Видео для верификации исполнителя |
| `StateClientVerif` | Документ для верификации заказчика |
| `StatePaymentReceipt` | Чек оплаты подписки |
| `StateAdminEditUser` | Редактирование профиля администратором |

### Вспомогательные данные FSM

Помимо состояния, FSM хранит произвольные значения (`SetVal` / `GetStr` / `GetInt64` / `GetBool`):

- `"lang"` — выбранный язык до регистрации
- `"title"`, `"description"`, `"category"`, `"budget_type"`, `"budget_from"`, `"budget_to"`, `"deadline"`, `"refs"`, `"urgent"` — данные создаваемой задачи
- `"respond_task_id"`, `"respond_msg"` — данные отклика
- `"review_task_id"`, `"review_executor_id"`, `"review_rating"` — данные отзыва
- `"pay_type"`, `"pay_amount"` — данные платежа
- `"edit_user_id"`, `"edit_field"` — данные редактирования (администратор)

## Локализация

**Расположение:** `internal/locales/locales.go`

### Архитектура

```go
// Все переводы — пары [RU, UZ]
var tr = map[string][2]string{
    "welcome_new": {"Добро пожаловать!", "Xush kelibsiz!"},
    // ...
}

type Loc struct{ lang string }

func New(lang string) *Loc   // создать локаль
func (l *Loc) T(key string, args ...interface{}) string  // перевод с аргументами
```

### Как работает `b.loc(tgID)`

```
b.loc(tgID)
  1. Проверяет FSM: есть ли ключ "lang"? → использует его
  2. Если нет → запрашивает users.GetLanguage(tgID) из БД
  3. Если не найден → возвращает "ru" (по умолчанию)
```

### Меню: обратный поиск

Кнопки Reply-клавиатуры (нижнее меню) идентифицируются **по тексту**, который зависит от языка. Для универсальной обработки используется `menuActions` map:

```go
var menuActions = map[string]string{
    "➕ Создать задачу": "create_task",
    "➕ Vazifa yaratish": "create_task",
    // ...
}

func MenuAction(text string) string  // текст кнопки → ключ действия
```

В `handleMenu` используется `switch locales.MenuAction(m.Text)` — работает для обоих языков.

### Добавить новый перевод

1. В `var tr` добавить запись:
   ```go
   "my_key": {"Русский текст %s", "O'zbek matni %s"},
   ```
2. Использовать в коде:
   ```go
   loc.T("my_key", argument)
   ```

## Важные особенности

**Уведомления на языке получателя.** Когда бот уведомляет пользователя (новая задача, выбор исполнителя, отклонение), он определяет язык каждого получателя отдельно через `b.loc(recipient.TelegramID)`.

**Безопасность администратора.** Все административные функции проверяют `b.isAdmin(tgID)` — сравнение с картой `adminIDs` из конфига.

**Редактирование сообщений платежа.** После решения по платежу бот **редактирует** (не отправляет новое) исходное сообщение у каждого администратора через `NewEditMessageText`, добавляя статус и имя проверившего.

**Срочные задачи.** Задача с `is_urgent = true` видна в ленте и уведомлениях только исполнителям с активной подпиской PRO.

**Лимиты откликов.** Без подписки исполнитель может откликнуться ограниченное количество раз (по умолчанию 5, настраивается через `FREE_RESPONSES`). Лимит хранится в `usage_limits` и уменьшается при каждом отклике.
